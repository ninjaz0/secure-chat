# SecureChat v0.2.6 Release Notes

SecureChat v0.2.6 is the next release after v0.2.5. It keeps the existing E2EE
wire protocol and attachment chunking model, removes the unfinished group chat
surface from all clients, and adds the first native Windows client source tree.

## Highlights

- macOS, iOS, Android and Windows now expose one-to-one chat and temporary
  sessions only. Experimental group APIs and UI entries were removed from the
  public client surface.
- Windows client source is available under `apps/windows/SecureChatWindows`,
  built with WinUI 3, Windows App SDK and C# P/Invoke over `secure_chat_ffi.dll`.
- Windows runtime secrets use Credential Manager/DPAPI through the Rust desktop
  runtime. Local SQLite, attachments, stickers and temporary files live under
  `%LOCALAPPDATA%\SecureChat`.
- File, image, sticker and burn-after-reading messages continue to use encrypted
  chunks sized below the relay HTTP payload limit.
- Rust lint and security cleanup: the workspace passes `cargo audit` and Clippy
  warning-as-error checks for the verified release candidate.

## Verification Commands

Verified locally on 2026-05-09:

```bash
PATH="$HOME/.cargo/bin:$PATH" cargo audit
PATH="$HOME/.cargo/bin:$PATH" cargo clippy --workspace --all-targets -- -D warnings
PATH="$HOME/.cargo/bin:$PATH" cargo test --workspace
SECURE_CHAT_VERSION=0.2.6 ./script/package_macos.sh
codesign --verify --deep --strict --verbose=2 dist/release/SecureChatMac.app
./script/build_ios.sh debug
ANDROID_HOME=/opt/homebrew/share/android-commandlinetools ./script/build_android.sh release
ANDROID_HOME=/opt/homebrew/share/android-commandlinetools dist/gradle/gradle-8.10.2/bin/gradle -p apps/android/SecureChatAndroid lintRelease
```

Final Android asset was repackaged after stripping generated native libraries
with the Android NDK `llvm-strip`, then verified again with `apksigner`.

Windows project files were statically validated with `xmllint`. Windows package
build command for a Windows 10 22H2+ or Windows 11 build host:

```powershell
.\script\build_windows.ps1 -Configuration Release -Version 0.2.6.0 -ArtifactVersion 0.2.6
```

## Release Assets

- `SecureChatMac-0.2.6.dmg`
- `SecureChatAndroid-0.2.6.apk`
- `SHA256SUMS.txt`

Windows MSIX/EXE packaging requires a Windows host with PowerShell, .NET,
Windows App SDK and MSVC tooling. The macOS release host validated project files
and attempted `cargo build --locked -p secure-chat-ffi --target
x86_64-pc-windows-msvc --release`, but could not produce the DLL because the
Windows SDK headers were unavailable (`windows.h` / `assert.h`). `pwsh` and
`dotnet` were also unavailable on this host, so no Windows binary asset is
included in this release.

## Signing Notes

- macOS DMG is ad-hoc signed unless `SECURE_CHAT_MACOS_SIGN_IDENTITY` and
  notarization are configured. Gatekeeper may still warn on first launch.
- Android APK is installable and signed by the release build script. The
  verified signer certificate SHA-256 is
  `2f930b160ddbde2abcfaee2da9f10cd370995e93d9c44385139a6bf33343e38e`.
  Use a real store/organization release keystore for public store-style
  distribution.
- Windows public distribution should use a trusted code signing certificate;
  local test certificates are suitable only for internal validation.
