# SecureChat v0.2.6 Windows Release Notes

SecureChat v0.2.6 adds the first native Windows client while keeping the v0.2.5
wire protocol, Rust runtime, relay behavior, and E2EE message model.

## Highlights

- New WinUI 3 + Windows App SDK client under
  `apps/windows/SecureChatWindows`.
- C# P/Invoke wrapper over `secure_chat_ffi.dll`, including app snapshot,
  bootstrap, relay settings, invites, contacts, groups, temporary sessions,
  attachments, burn messages, stickers, receive, push registration and P2P probe
  diagnostics.
- Windows Credential Manager/DPAPI support for desktop runtime secrets.
- `%LOCALAPPDATA%\SecureChat` app data directory for SQLite, attachments,
  stickers and temporary files.
- Feature parity with v0.2.5 chat capabilities: contact nickname editing,
  local strong delete, file/image/sticker sending, burn-after-reading messages,
  Unicode emoji and smart auto-scroll.
- MSIX packaging script with local test certificate generation and SHA-256
  checksum output.

## Verification Commands

```powershell
.\script\build_windows.ps1 -Configuration Release -Version 0.2.6.0 -ArtifactVersion 0.2.6
cargo check -p secure-chat-ffi --target x86_64-pc-windows-msvc
cargo test --workspace
```

## Release Assets

- `SecureChatWindows-0.2.6.msix`
- `SecureChatWindows-0.2.6.msix.sha256`
- `SecureChatWindows-TemporaryKey.cer` when a local test certificate is used

The test certificate path is suitable for local/internal validation only.
Public distribution should use a trusted code signing certificate and publish
the certificate thumbprint with the release.
