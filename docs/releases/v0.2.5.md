# SecureChat v0.2.5 Release Notes

SecureChat v0.2.5 expands the chat surface across macOS, iOS, and Android while
keeping the existing relay and E2EE envelope model. This release is still a
self-hostable prototype and has not had third-party cryptographic review.

## Highlights

- Contact nickname editing and local strong contact deletion.
- File and image sending with encrypted local metadata and transfer state.
- Local sticker image/GIF import and sticker sending.
- Burn-after-reading messages with local destruction and best-effort encrypted
  destroy notices.
- Unicode emoji input/display through the system keyboard or input method.
- Smarter chat auto-scroll: new messages follow only when the user is already
  near the bottom; history review shows a lightweight new-message button.

## Compatibility

- Old text rows continue to render through the `body` fallback.
- macOS, iOS, and Android use the same Rust FFI/JNI runtime and relay protocol.
- Attachments do not have a product-level hard limit, but relay chunk size,
  queue capacity, disk, network, and platform resources still apply.

## Verification Commands

```bash
PATH="$HOME/.cargo/bin:$PATH" cargo test -p secure-chat-core -p secure-chat-desktop -p secure-chat-ffi
PATH="$HOME/.cargo/bin:$PATH" cargo test --workspace
PATH="$HOME/.cargo/bin:$PATH" cargo check -p secure-chat-ffi
PATH="$HOME/.cargo/bin:$PATH" cargo audit --deny warnings
SECURE_CHAT_VERSION=0.2.5 ./script/package_macos.sh
./script/build_ios.sh debug
ANDROID_HOME=/opt/homebrew/share/android-commandlinetools \
ANDROID_SDK_ROOT=/opt/homebrew/share/android-commandlinetools \
./script/build_android.sh release
```

distribution, so Gatekeeper may require Control-click Open. Official public
distribution still requires Developer ID signing and notarization.
