# SecureChat v0.2.7 Release Notes

SecureChat v0.2.7 is a maintenance release after v0.2.6. It keeps the current
one-to-one chat, temporary sessions, attachment chunking, stickers,
burn-after-reading messages, Unicode emoji support, contact management and
Windows source tree intact, while fixing the macOS app startup crash reported
after the group-chat surface was removed.

## Highlights

- Fixed macOS launch failure caused by the app binary loading
  `libsecure_chat_ffi.dylib` from the developer build directory instead of the
  bundled `Contents/Frameworks` copy.
- macOS release and local run scripts now rewrite the FFI dynamic library load
  command to `@rpath/libsecure_chat_ffi.dylib`, so the app and Rust FFI dylib are
  shipped as a self-contained pair.
- The macOS app bundle is verified with `otool -L` and `codesign --verify` before
  publishing.
- Version metadata is aligned to `0.2.7` across Rust crates, macOS, iOS, Android
  and Windows project files.

## Verification Commands

Verified locally on 2026-05-10:

```bash
PATH="$HOME/.cargo/bin:$PATH" cargo test --workspace
PATH="$HOME/.cargo/bin:$PATH" cargo audit --deny warnings
./script/package_macos.sh
otool -L dist/release/SecureChatMac.app/Contents/MacOS/SecureChatMac
codesign --verify --deep --strict --verbose=2 dist/release/SecureChatMac.app
ANDROID_HOME=/opt/homebrew/share/android-commandlinetools ./script/build_android.sh release
ANDROID_HOME=/opt/homebrew/share/android-commandlinetools gradle -p apps/android/SecureChatAndroid lintRelease testReleaseUnitTest
./script/build_ios.sh debug
```

The macOS app installed to `/Applications/SecureChatMac.app` was launched and
confirmed to show the main SecureChat window.

## Signing Notes

- macOS DMG is ad-hoc signed unless `SECURE_CHAT_MACOS_SIGN_IDENTITY` and
  notarization are configured. Gatekeeper may still warn on first launch.
- Android APK is installable and signed by the release keystore used for prior
  SecureChat APK releases. The verified signer certificate SHA-256 is
  `2f930b160ddbde2abcfaee2da9f10cd370995e93d9c44385139a6bf33343e38e`.
  Use a real store/organization release keystore for public store-style
  distribution.
